## 1 Feature Summary

### What / Why

- **Goal**：解决多绑定（multi-binding）轴场景下轴标签被“分离/散开”导致的可读性问题，让用户无需写 Script 也能一键把标签放到对侧。
- **User entry**：选中轴标签区域 → 右键 → `Properties` → `Label` → 勾选 `Labels on Opposite Side`。
- **Key behavior**：
  - X 轴：bottom ↔ top
  - Y 轴：left ↔ right
  - 多绑定轴：勾选后标签应“合并到对侧”，不再出现分离渲染；取消勾选恢复原侧
  - 多轴/双轴：每条轴独立生效，互不串扰；避免左右轴标签重叠或错位

### Scope & Non-scope

- **In scope**：
  - Axis-based（标准 X/Y 坐标轴）：Bar/Column/Line/Area/Scatter/Bubble/Step Line/Waterfall/Pareto
  - Multi-axis（双 Y 轴/组合图）：Bar（双轴）、Line（双轴）、Area（双轴）、Combo
  - Financial：Candle/Stock（行为同标准 X/Y，主要验证不回归）
- **Out of scope（不应支持/不应暴露入口）**：
  - Polar：Radar（测试结论：极坐标不支持）
  - Partition：Pie/Donut/Treemap/Sunburst（无轴）
  - Network：Network/Hierarchy（无轴）
  - Geographic：Map/Density Map（测试结论：不支持）
  - None：Tree/Network/Circular Network/Scatter Contour/Contour Map 等（无严格意义 axis，不会弹 Axis Properties）
  - 其他特殊：Funnel、Gantt（需单独确认是否会暴露入口，若暴露按探索性测试处理）

## 2 Functional Rules（规则化描述，避免只写步骤）

> 用“默认 + 变化 + 约束 + 优先级”的方式描述规则，便于写用例和定位回归。

- **Default behavior**：checkbox 默认未勾选；轴标签显示在默认侧（X=bottom，Y=left；双轴按各自默认侧）。
- **Toggle / config behavior**：
  - 勾选：该轴标签移动到对侧（X：top；Y：right）
  - 取消：恢复默认侧
  - 保存/刷新/重新打开对话框：checkbox 状态正确回显；渲染位置一致
- **Multi-object interaction（多轴/多绑定）**：
  - 多绑定轴：勾选后标签应统一到对侧（不分离）
  - 同一图表 X/Y 分别配置：互不影响
  - 双轴（Left/Right）：各自勾选后不应重叠；两轴同时勾选也不应混乱
  - 风险点（来自测试-分析）：当存在/设置 second axis 时，可能出现“属性不应支持但仍可操作/操作无效/轴丢失/轴乱”
- **Priority rules（UI vs Script）**：
  - 若 Script 已通过 `AxisSpec.setAxisStyle(AxisSpec.AXIS_SINGLE2 / AXIS_DOUBLE2 ...)` 设置到对侧：**Script 优先于 UI**，UI 勾选应无效且不引发状态叠加异常（测试-分析结论）。
- **Error handling / guardrails**：
  - 不支持类型：不应暴露 Axis Properties / Label 页或不包含该 checkbox
  - 若误暴露：勾选不应崩溃、不乱码、不导致轴布局错乱
  - Pareto：本身双 Y 轴，按测试-分析结论 **不应该支持**（当前有 Bug #74009）

## 4 Test Focus（只列必须测的路径）

### P0 - Core Path（必测主路径）

- **Core scenario(s)**：axis-based（Bar/Column）下 X 轴与 Y 轴分别勾选/取消，渲染正确（对侧/恢复）。
- **Persist & reload**：保存 Dashboard → 刷新/重新打开 → 导出 vsml → 重新导入，checkbox 回显与渲染一致（测试-分析：需通路验证状态保存）。
- **UI state echo**：同一图表 X/Y 分别打开 Properties，各自 checkbox 独立回显，互不影响。

### P1 - Functional / Boundary（高风险功能与边界）

- **Multi-binding / multi-axis / mixed modes**：
  - 多绑定 X/Y 轴：勾选后标签不分离
  - 双轴/Combo：Left/Right 各自勾选、两轴同时勾选不混乱
  - second axis 相关限制：设置 second axis 后该属性应禁用/不支持（见 Issue #74047 等）
- **Not supported / negative paths**：Radar/Map/Pie/Network 等不应暴露入口；Pareto 不应支持（Bug #74009）。
- **Locale**：验证非英文 locale 下 checkbox 文案显示正常（测试-分析：初步 ok）。

### P2 - Extended（按需抽样）

- **Export / Print / Embedded**：PDF/图片导出与嵌入 Report 场景抽样验证（测试-分析：通路验证状态保存）。
- **Performance**：无需专项；抽样多绑定/双轴对话框操作与渲染无明显性能问题。
- **Backward compatibility**：旧 vsml（无 `labelOnSecondaryAxis`）导入默认 false；若旧脚本设置了 `AXIS_SINGLE2/DOUBLE2`，验证 Script 优先且 UI 不造成异常（测试-分析：可忽略 UI 同步，按 Script 优先验证稳定性）。

## 6 Special Testing（仅当涉及时填写）

### Security

- 无新增外部输入面；常规回归即可。

### Performance

- 无专项；抽样多绑定/双轴下打开对话框与切换不明显卡顿。

### Compatibility

- **Old config**：旧 vsml 不含 `labelOnSecondaryAxis` 时默认 false，行为与旧版一致。
- **Round-trip**：勾选后保存/导出/导入不丢状态（至少覆盖 Bar/Column 各 1）。

### Script / Programmatic Support

- **API(s)**：`chart.getAxis("Bottom/Left/Right").setAxisStyle(...)`、`AxisSpec.AXIS_SINGLE2 / AXIS_DOUBLE2`。
- **UI vs Script priority**：测试-分析结论：**Script 优先，UI 设置无效**（需验证“无效但稳定”）。
- **Gotchas**：关注多次切换/多轴下是否出现轴丢失、轴乱、属性无效但可操作等（见 Bug/Issue 列表）。

### Localization

- **Strings / keys**：`Labels on Opposite Side`。
- **Fallback behavior**：无翻译时应英文 fallback，不显示 key 原文/乱码（测试-分析：初步 ok）。

### Docs / API

- 若对外脚本文档涉及 `getAxisStyle()/setAxisStyle()`，补充 UI 与 Script 优先级说明（至少在测试备注中固化）。
